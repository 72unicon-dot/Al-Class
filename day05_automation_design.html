업무 프로세스 자동화 설계

무엇을 자동화할 것인가? 설계가 반이다.

자동화 로드맵 그리기

1. 업무 분해 (Deconstruction)

현재 하고 있는 일을 가장 작은 단위(Unit Task)로 쪼갭니다.
예: 이메일 확인 -> 첨부파일 다운로드 -> 폴더 열기 -> 저장 -> 엑셀 입력

2. 패턴 식별 (Pattern Recognition)

"반복적"이고 "규칙적"인 업무를 찾습니다. 창의성이나 판단이 필요한 업무는 제외합니다.

3. 도구 매핑 (Tool Mapping)

각 단계에서 데이터를 주고받을 수 있는 도구(Gmail, Slack, Sheets 등)를 확인합니다.

자동화 적합성 체크리스트

반복 빈도가 높은가? (매일/매주)
규칙(Rule)이 명확한가?
입력 데이터가 디지털 형태인가?
오류 발생 시 리스크가 낮은가?

강의 요약 및 다음 단계

반복적인 업무 흐름을 자동화하여 창의적인 일에 더 집중할 수 있는 시간을 확보하세요.
